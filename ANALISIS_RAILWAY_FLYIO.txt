⚠️ PROBLEMA COMÚN EN TODAS: SQLite se borra

═══════════════════════════════════════════════════════════════════════

En plataformas cloud (Render/Railway/Fly.io):
  ❌ El disco es "efímero" (temporal)
  ❌ Cada deploy BORRA tu base de datos SQLite
  ❌ Necesitas migrar a PostgreSQL/MySQL

SOLUCIONES:

A) Migrar a PostgreSQL:
   - Railway: PostgreSQL gratis incluido
   - Fly.io: PostgreSQL $5/mes
   - Render: PostgreSQL gratis (básico)

B) Usar volumen persistente:
   - Fly.io: +$0.15/GB/mes
   - Railway: Incluido en plan
   - Render: No disponible en plan gratis

C) Usar red local + SQLite:
   - Tu solución actual
   - Gratis, datos permanentes
   - Solo requiere VPN Walmart

🔧 CÓMO MIGRAR (si decides hacerlo):

═══════════════════════════════════════════════════════════════════════

RAILWAY:
  1. Crea cuenta: https://railway.app
  2. "New Project" → "Deploy from GitHub"
  3. Conecta tu repo
  4. Agrega PostgreSQL (gratis)
  5. Modifica código para usar PostgreSQL
  6. Deploy automático
  7. Paga $5/mes

FLY.IO:
  1. Instala Fly CLI: https://fly.io/docs/hands-on/install-flyctl/
  2. fly auth signup
  3. fly launch (en carpeta del proyecto)
  4. Configura PostgreSQL
  5. Modifica código
  6. fly deploy
  7. Paga ~$3/mes

═══════════════════════════════════════════════════════════════════════

⚠️ ADVERTENCIA: Migración NO es trivial

═══════════════════════════════════════════════════════════════════════

Cambios necesarios:

1. Base de datos: SQLite → PostgreSQL
   - Reescribir create_db.py
   - Migrar datos existentes
   - Cambiar conexiones

2. Archivos estáticos:
   - Configurar almacenamiento cloud
   - O usar Cloudinary/S3

3. Variables de entorno:
   - DATABASE_URL
   - SECRET_KEY
   - Etc.

Tiempo estimado: 2-4 horas
